# LifeThreads Launch Checklist

## Product Readiness

- Core memory creation works with one or multiple photos.
- Edit memory works for title, story, date, location, category, feeling, cover photo, and gallery photos.
- Wall, Timeline, and Map modes are stable.
- Text notes, nails, ropes, and connection reasons are understandable without explanation.
- Backup export and import work on at least 3 physical Android devices.
- Clear all data works and does not leave copied photos behind.
- Premium free limit blocks new memories at 30 memories.
- Premium unlock and restore are verified with Google Play test accounts.

## Store Readiness

- App name: LifeThreads.
- Package: `dev.gkcoding.lifethreads`.
- Signed release AAB built with production keystore.
- Privacy policy live at `https://gkcoding.dev/lifethreads/privacy`.
- Google Play listing added from `docs/play_store_content.md`.
- At least 6 polished screenshots uploaded.
- Feature graphic uploaded.
- Data Safety form matches privacy policy.
- Content rating questionnaire completed.
- Category set to Lifestyle.
- Managed product created: `lifethreads_premium_lifetime`.
- Closed beta testers added.

## Closed Beta Exit Criteria

- Minimum 10 testers complete the checklist.
- No data-loss bug remains open.
- No critical crash remains open.
- At least 5 testers create 5+ memories.
- At least 3 testers test backup export/import.
- At least 3 testers test Premium purchase/restore using Play test accounts.
- Feedback confirms the app feels emotional, not just functional.

## Launch Day

- Publish production release or staged rollout.
- Post first demo video.
- Add landing page link to GK Coding site navigation or footer.
- Share Play Store link on LinkedIn.
- Ask beta testers to give private feedback first, not public reviews.
- Track installs, crashes, store listing conversion, and feedback themes.

## First Week After Launch

- Fix crash/data-loss bugs first.
- Respond to all useful feedback.
- Improve screenshots if conversion is weak.
- Publish 3 short videos.
- Avoid adding big new features before stability is proven.
