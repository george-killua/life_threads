# LifeThreads 30-Day Marketing Plan

## Goal

Validate whether LifeThreads creates emotional value strong enough for people to keep using it and pay for a one-time premium unlock.

Primary metric:

```text
Users create 5+ memories and return within 7 days.
```

Secondary metrics:

- Play Store visits to installs
- installs to first memory created
- first memory to 5 memories
- premium screen views
- premium unlock purchases
- qualitative feedback: "I would keep personal memories here"

## Week 1: Beta Proof

Day 1:
- Upload closed beta build.
- Invite 10-20 trusted testers.
- Send tester checklist.
- Ask for emotional feedback, not only bug reports.

Day 2:
- Watch crash reports and beta feedback.
- Fix blocking onboarding or memory creation issues.

Day 3:
- Record first raw demo: travel memory wall.
- Capture screenshots for Play listing.

Day 4:
- Ask testers: "What confused you in the first 2 minutes?"
- Fix one onboarding or wall clarity issue.

Day 5:
- Record couple/family memory demo.
- Prepare landing page assets.

Day 6:
- Verify backup/export/import flow.
- Verify premium screen and restore flow.

Day 7:
- Decide: continue beta, or prepare production staged rollout.

## Week 2: Public Launch Preparation

Day 8:
- Publish landing page at `gkcoding.dev/lifethreads`.
- Add privacy policy route.

Day 9:
- Create 6 final screenshots.
- Create feature graphic.

Day 10:
- Final Play listing copy check.
- Confirm Data Safety answers.

Day 11:
- Create 3 TikTok/Reels videos from scripts.

Day 12:
- Post LinkedIn build-in-public story.

Day 13:
- Prepare staged rollout, 10% if Play allows.

Day 14:
- Launch Android production or widen closed beta.

## Week 3: Content and Feedback

Day 15:
- Post Video 1: "Your gallery stores photos. LifeThreads connects memories."

Day 16:
- Read reviews/private feedback.
- Fix the most repeated confusion.

Day 17:
- Post Video 2: couple/family timeline.

Day 18:
- Test store conversion: icon, screenshots, first line of description.

Day 19:
- Post Video 3: travel wall/map.

Day 20:
- Ask testers for permission to quote private feedback later. Do not invent testimonials.

Day 21:
- Patch release if needed.

## Week 4: Monetization Validation

Day 22:
- Review premium screen views and purchases.

Day 23:
- Improve premium screen if people hit the limit but do not unlock.

Day 24:
- Post Video 4: backup/export/privacy.

Day 25:
- Create a simple comparison post: gallery vs LifeThreads.

Day 26:
- Ask 5 users why they would or would not pay €4.99.

Day 27:
- Fix the strongest premium objection if it is product-related.

Day 28:
- Prepare next update roadmap.

Day 29:
- Post Video 5: memory room themes.

Day 30:
- Decision review:
- Continue Android growth.
- Improve retention.
- Start iOS only if retention is promising.
